# Test assets

Every asset in this directory documents its provenance and license here. Test
assets live outside the built distribution — the osr-forge wheel ships no game
content.

## minimod/

An original mini-module, *The Root Cellar of Old Wenna*, authored for this
repository as a test asset and dedicated to the public domain under
[CC0 1.0](https://creativecommons.org/publicdomain/zero/1.0/).

- `minimod.pdf` — 5 pages. Pages 1–3 and 5 carry a text layer (keyed areas,
  stated room dimensions, monster names); page 4 is a raster map image with no
  text layer, exercising the scanned-module path.
- `encrypted.pdf` — a 1-page password-protected PDF exercising the
  encrypted-source error path. The password is `minimod-owner-password` (it is
  never used; the tests only assert that opening without it fails).
- `pages/` — the exact page renders and text layers the extraction fixtures
  were recorded against (phase 1's recording session).
- `fixtures/` — recorded `survey` and `content` fixtures from a real
  `preprocess → survey → content` run over `minimod.pdf`.
- `expected/` — the golden stage caches (`survey.json`,
  `areas.<dungeon>.<level>.json`) that run's stage functions wrote; the
  pipeline-replay test pins them byte-for-byte.

The PDFs are generated by `tools/minimod/generate_minimod.py` (reportlab +
Pillow). Regenerate from the repo root with:

```sh
uv run tools/minimod/generate_minimod.py
```

Regenerating changes the PDF bytes (creation timestamps); nothing byte-compares
the committed PDFs, so that's harmless — but don't regenerate without a reason.

**Fixture coupling:** regenerating `minimod.pdf`, bumping pypdfium2 or Pillow,
or editing an extraction prompt or schema strands `pages/`, `fixtures/`, and
`expected/` together — request fingerprints hash the page bytes and the prompt
text. Re-record the whole set via the minimod session commands in
`tools/extract/README.md`; never rebuild fixture requests from fresh renders.
